About Me

Hey there!
I'm Jayant Yadav, an undergraduate student at Dr. Shyama Prasad Mukherjee International Institute of Information Technology, Naya Raipur — more commonly known as IIIT-NR — where I'm diving deep into Electronics and Communication Engineering.

While many around me are treading the well-worn path of software development, I’ve opted to take the low-level route — one filled with hardware, logic gates, and real-world signals. From the smell of burnt flux to the click of a relay, I find satisfaction in building things that exist beyond the screen.

I have a strong interest in PCB design and prototyping, which naturally pushed me into exploring various sensors and how they breathe life into IoT and robotics. I’ve worked with the classic Arduino ESP32 DevKit, and I’m currently eyeing the LGT8F328P — kind of like a Nano, but with more power. Every schematic I draw and board I test is one more step toward bridging abstract theory with hands-on tech.

Lately, I’ve been working with Digital Logic Design and Hardware Description Languages (HDLs), working on a passion project: designing and simulating my own 16-bit (may scale it to 32-bit) CPU from scratch. It’s not just about writing modules — it’s about digging into the foundations of computation, understanding how everything clicks at the gate level, and turning that into a functional design.

I also daily-drive Linux (Fedora) — not out of some pretentious "ricing" phase, but because it gets out of my way and lets me work.
I also collect stamps and swim.

I believe in learning by building, failing early, and pushing what an undergrad can accomplish without sugarcoating the grind. If it’s low-level, timing-sensitive, or looks like a circuit diagram had a crossover with an instruction set — chances are, I’m working on it.

CPU Design

Though an ongoing project, this project is aimed towards designing a 16 bit (upscale to 32 bit) CPU which can run basic tasks.
The whole design is using the universal gate NAND, (NOR can also be used) the first basic designs are being set up on HDL code bits written on VS codeand ran using simulators
The design includes:
- ALU (Arithmetic Logic Unit) for basic arithmetic operations
- Control Unit for instruction decoding and execution
- Register File for storing temporary data

Central Control System MK1: Pale Raven

This ongoing project builds a remotely controlled robot using an ESP32-hosted GUI. User commands are relayed to an Arduino Uno via 1:1 GPIO signaling to drive two DC motors. An ultrasonic sensor provides basic obstacle detection.

The focus is on achieving low-latency, reliable communication between microcontrollers,separating logic and actuation layers. With delays only introduced by the ESP32 web interface.
